# WeekFlow

WeekFlow — персональная рабочая панель недели для человека, который одновременно ведёт несколько проектов, рабочие и личные задачи.

Главная идея продукта — за 10 секунд показать всю неделю: что нужно сделать, какие дни перегружены, к каким проектам относятся задачи, что просрочено и что требует повышенного внимания.

> Проект находится в активной разработке. Сейчас реализован статический адаптивный интерфейс главного экрана на вымышленных демонстрационных данных. Сохранение и редактирование задач появятся на следующих этапах.

## Основные возможности

В текущей версии доступны:

- обзор всех семи дней недели на одном экране;
- спокойная индикация загрузки каждого дня;
- карточки однодневных задач;
- непрерывные полосы многодневных задач;
- постоянные цвета проектов и отдельные обозначения приоритета;
- заметные текстовые состояния просроченных и выполненных задач;
- отдельная зона задач на сегодня;
- демонстрационное быстрое добавление без сохранения данных;
- компактные точки входа во «Входящие» и «Проекты»;
- адаптивная компоновка для Mac и экранов телефона шириной 360–390 пикселей;
- русский интерфейс и базовая клавиатурная доступность.

Все названия проектов и задач в репозитории полностью вымышлены.

## Технологии

- [React](https://react.dev/) и React DOM;
- [TypeScript](https://www.typescriptlang.org/) в строгом режиме;
- [Vite](https://vite.dev/) для разработки и сборки;
- [date-fns](https://date-fns.org/) для будущей календарной логики;
- CSS Modules и CSS-переменные;
- [Vitest](https://vitest.dev/) и React Testing Library;
- ESLint и Prettier.

В проекте намеренно нет Redux, React Router, UI-фреймворка, сервера, базы данных и библиотек авторизации.

## Запуск проекта

### Требования

- Node.js `20.19+` или `22.12+`;
- npm.

### Установка и запуск

Клонируйте репозиторий, перейдите в каталог проекта и выполните:

```bash
npm ci
npm run dev
```

После запуска приложение будет доступно по адресу, который покажет Vite, обычно [http://localhost:5173](http://localhost:5173).

### Доступные команды

| Команда                | Назначение                                       |
| ---------------------- | ------------------------------------------------ |
| `npm run dev`          | Запустить локальный сервер разработки            |
| `npm run build`        | Проверить TypeScript и собрать production-версию |
| `npm run typecheck`    | Проверить типы без сборки                        |
| `npm run lint`         | Проверить исходный код ESLint                    |
| `npm run test`         | Запустить тесты один раз                         |
| `npm run format:check` | Проверить форматирование Prettier                |
| `npm run format`       | Отформатировать файлы Prettier                   |
| `npm run preview`      | Локально открыть production-сборку               |

Перед завершением этапа разработки должны успешно проходить `typecheck`, `lint`, относящиеся к этапу тесты и `build`.

## Текущее состояние

Завершены два первых этапа:

1. Настроен рабочий каркас React, TypeScript и Vite с проверками качества.
2. Реализован статический интерфейс главного экрана с недельным обзором, блоком «Сегодня», быстрым добавлением и адаптивным отображением.

Текущие данные расположены отдельно в `src/demo/` и используются только для демонстрации интерфейса. Приложение не отправляет данные во внешние сервисы и пока не записывает их в браузерное хранилище.

## Известные ограничения

- задачи, проекты, даты и выбранная неделя пока статичны;
- кнопки переключения недели, «Входящие» и «Проекты» ещё не открывают рабочие представления;
- быстрое добавление показывает демонстрационную реакцию, но не создаёт запись;
- отсутствуют создание, редактирование, перенос, завершение и удаление задач;
- введённые пользователем данные не сохраняются в `localStorage` и сбрасываются после обновления страницы;
- редакторы задач и проектов, маршрутизация и полноценная бизнес-логика ещё не реализованы;
- приложение предназначено для одного пользователя и не поддерживает синхронизацию между устройствами.
